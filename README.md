# KION2

Подбор и обучение моделей, которые по позам (pose estimation) актёров в видео определяют их действия на предложенном датасете.

## Описание проекта

Проект содержит несколько моделей определения поз по видеоряду: YOLOv8-Pose-N,
YOLOv8-Pose-S, YOLOv26-Pose-N. Для определения действий актеров в кадре была взята
готовая модель st-gcn, а также созданы и натренированы 2 своих модели (одна полностью
на сверточных слоях, вторая на комбинации сверточных и рекурсивных слоев).  
Проект содержит ряд скриптов для тренировки и тестирования моделей. Основной скрипт 
анализа видео - action_detector.py. Скрипт принимает на вход путь до видеофайла,
путь до файла конфигурации в формате yaml, опционально можно передать путь до папки
сохранения результатов работы скрипта и флаг подробного логирования процесса.
В проекте реализован простейший трекер актеров на базе IoU между прямоугольниками
обнаруженных людей во время обработки каждого отдельного кадра.
Результатом работы основного скрипта является json файл со списком обнаруженных поз с
идентификаторами актеров из трекера и значениями уверенности модели, а также список
действий с идентификаторами актеров и значениями уверенности модели.  
Позы в выходном файле представлены списком из 30 ключевых точек, содержащих координаты 
на экране в пикселях и уверенность модели в обнаружении положения каждой точки.

В проекте приложен файл настроек работы основного скрипта config.yml вида:
```antlrv4
act_frames: 120
batch_size: 16
multimodal_model: TST
pose_action_model: LSTMSkeletonNet
pose_ext_model: YOLOv26-Pose-N
video_decimation: 1
pose_ext_th: 0.1
```
где act_frames - число кадров для определения действия,  
 batch\_size - размер пакета данных при тестировании или предсказании,  
 multiple_modality_models - тип мультимодальной модели определения действий 
(в данном проекте не было реализовано из-за высокой вычислительной сложности),  
pose-action-model - используемая нейронная сеть для определния поз,  
 pose-ext-model - используемая нейронная сеть для определения действий по позам,  
 video decimatinon - коэффициет уменьшения размера обрабатываемого видеофрагмента за счет пропуска кадров,  
 poset ext threshhold - минимальная вероятностное значение детекторa поз и детектора действий 
кандитата для его включению в итоговый результат.

## Разворачивание окружения

Было использовано окружение conda с python3.11. Для работы pytorch на мощностях видеокарты было установлено 
окружение NVIDIA CUDA 11.8. 

### Последовательность установки

#### Установка в виртуальное окружение

Установите зависимости:
   
   **Установите PyTorch с нужной версией CUDA:**
   
   ```bash
   # Проверьте версию CUDA: 
   nvidia-smi
   
   # Для CUDA 13.0:
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu130
   
   # Для CUDA 12.8:
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128
   
   # Для CUDA 12.6:
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu126
   
   # Для CUDA 12.1:
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121
   
   # Для CUDA 11.8:
   pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118
   
   # Для CPU (если нет GPU):
   pip install torch torchvision
   ```
   
   **Важно:** 
   - Сначала установите PyTorch, затем остальные зависимости
   - Для актуальной информации о совместимых версиях используйте официальный сайт: https://pytorch.org/get-started/locally/


```bash
pip install -r requirements.txt
```

Для работы необходим ffmpeg
Установка Ubuntu Linux: 

```
sudo apt update
sudo apt install ffmpeg
```
Установка Windows10/11: 

```
winget install ffmpeg
```

#### Установка в Docker

Необходимое ПО:

**Docker Desktop** - основная платформа контейнеризации  
**NVIDIA Container Toolkit** - расширение для работы с GPU (устанавливается отдельно)  
**NVIDIA GPU драйверы** - драйверы видеокарты (устанавливаются отдельно)  
**CUDA Toolkit** - библиотеки для работы с GPU (обычно идет с драйверами)

#### Создание образа
Запустите Docker Desktop.
В папке проекта в терминале выполните:  
`docker build -t kion-app-cuda .` где:
- `docker build` - команда для создания Docker образа
- `-t kion-app-cuda` - задает имя (тег) образа
- `.` - указывает на текущую папку с Dockerfile


#### Запуск проекта в докер:
`docker run --rm --gpus all -v "${PWD}:/app" kion-app-cuda action-detector.py in1.mp4 --output_dir output_test --verbose --config_path config.yaml`
- `python` - интерпретатор Python
- `action_dector.py` - основной скрипт приложения
- `in1.mp4` - параметр с именем видеофайла
- `--output_dir output_test` - параметр с папкой для сохранения результатов
- `--verbose` - логирование выполнения программы
- `--config-path config.yml`- конфигурационный YAML  
  

## Отчеты

Для визуализации результатов (графики, отчеты) необходимо использовать блокнот Jupiter "visualization.ipynb".  
Для визуализации результатов работы на самом видео используется скрипт "tests/json_and_video.py". 
Скрипт получает на вход пути до видеофайла, файла json с результатами работы скрипта распознания действий "action_detector.py" 
и файла видео с результатами визуализации. Также скрипт принимает на вход параметры --show (включает отображение видео с визуализацией
в процессе работы скрипта), --debug-joints, включающий подписи суставов на видео.
Пример запуска:  
```antlrv4
python -m tests.json_and_video --video "C:/input_video.mp4" --json "C:/output/results.json" --show --out "tests/output/output_video.mp4"

```
Результатом работы скрипта является видеофайл с размеченными суставами людей, описанными прямоугольниками вокруг людей и 
подписи распознанных действий с их вероятностями.

## Запуск пайплайна «видео → действие»

- `tests/predict_single.py` — полный прогон: видео → позы  →  Top‑5 действий. Запуск из корня: `python tests/predict_single.py path/to/video.mp4` `tests/`

Скрипт анализирует короткий видеофайл с одним действием и выдает результат в виде 5 наиболее вероятных действий.

## Вспомогательные скрипты (анализ и отладка)

- `tests/tools/inspect_labels.py` — быстрый осмотр датасета Kinetics-skeleton или совместимого `.npy + .pkl`: баланс классов, форма `(N, 3, T, 18, M)`, статистика по каналам, визуализация нескольких кадров первого клипа.
- `tests/tools/show_inference_input.py` — показывает тот же numpy, который подаётся в ST-GCN при инференсе (через OpenPose JSON или наш JSON от `VideoProcessor`), печатает статистику по каналам и рисует скелеты.
- `tests/tools/test_stgcn_on_openpose_data.py` — прогоняет предобученную ST-GCN по папке с OpenPose `*_keypoints.json` для одного ролика, выводит Top‑5 предсказаний.
- `tests/tools/validate_stgcn_npy_pkl.py` — оффлайновая валидация ST-GCN на готовом `.npy` (+ `.pkl` с метками): считает Top‑1/Top‑5 accuracy или выводит предсказания для ручной проверки.
- `tests/test_lstm_gcn.py` - скрипт проверки работы собственной модели на базе сверток и рекурсивных слоев на данных поз.
- `tests/train_lstm_gcn.py` - скрипт тренировки собственной модели на базе сверток и рекурсивных слоев. Использует файл конфигурации обучения "tests/lstm_params.yml".
- `valid_proj.py` - скрипт валидации всего проекта (получения метрик качества) на датасете NTU RGB-D. Пример запуска:  
- `python -m tests.valid_proj "C:/NTU-RGB+D 120/Датасет" tests/config.yml `  
Скрипт получает на вход путь директории с видеофайлами NTU RGB-D датасета и путь до файла конфигурации проекта.

# Результаты работы

В рамках хакатона были реализованы модели распознания поз YOLOv8-Pose-N, YOLOv8-Pose-S, YOLOv26-Pose-N, 
модель распознания действий ST-GCN с двумя вариантами весов: на датасете kinetics400 - STGCN_model_kinetics, на датасете 
NTU RGB-D - STGCN_model_rgbd (этот вариант был обучен членами команды). Также членами команды были обучены
модели распознавания действий по скелетным данным собственной архитектуры: на основе сверточных слоев - Conv3dNet, на основе
сверточных слоев и рекурсивных слоев - LSTMSkeletonNet.  
Подробный отчет по получению метрик разных комбинаций моделей извлечения поз и моделей распознаия действий
приведен в документе [Валидация проекта](docs/evaluation_report.md)
