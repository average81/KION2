# Результаты тестирования моделей распознавания поз и действий по последовательности поз.

Визуализация работы проекта с видеофайлом выглядит примерно так:

![Пример работы](potter.gif)  

Метрики качества разных моделей в рамках проекта (суммарные метрики):

| модель распознавания действий | accuracy | f1     | precision | recall | Примечания                                                                                   |
|-------------------------------|----------|--------|-----------|--------|----------------------------------------------------------------------------------------------|
| STGCN_model_kinetics          |          |        |           |        |                                                                                              |
| STGCN_model_rgbd              | 59.69%   | 58.98% | 59.69%    | 59.09% | Обучена и тестирована на 2D координатах, совпадающих как в формате OpenPose, так и NTU RGB-D |
| Conv3dNet                     | 85.78%   | 85.78% | 86.37%    | 85.78% | Обучена и тестирована на 3D координатах                                                      | 
| LSTMSkeletonNet               | 57.53%   | 57.62% | 57.85%    | 57.76% | Обучена и тестирована на 2D координатах NTU RGB-D                                            |
| MLP                           | 39.72%   | 37.88% | 39.07%    | 39.72% | Обучена и тестирована на 3D координатах                                                      |

Время обработки видеофайла в формате FullHD длительностью 2 часа комбинацией моделей YOLOv26-Pose + ST-GCN Kinetics 
занимает 33 минуты на стенде Ryzen7 5700 + Nvidia RTX3060 12Gb.

## Анализ полученных метрик.
Обучение моделей на 2D данных (координатах на экране) дает значительно хуже результаты, чем на 3D данных.
ST-GCN заявляется с точностью до 88% на 3D данных, однако на 2D данных ее не удается обучить даже до 60%.

Сравнение ряда моделей распознавания действий и результаты их тренировок приведены в блокноте Hakaton_model_comparison.ipynb.
